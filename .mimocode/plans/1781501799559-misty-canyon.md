## PHASE 2: CRITICAL PERFORMANCE FIXES (Day 2-4)

### 2.1 Server Component Migration (Biggest Impact)
**Goal**: Convert 24/27 client components back to server components where possible.

**Priority pages to convert:**
| File | Current | Strategy |
|------|---------|----------|
| `app/pricing/page.tsx` | `"use client"` | Remove — fully static, zero interactivity |
| `app/(customer)/restaurants/page.tsx` | `"use client"` | Server component + client island for city filter |
| `app/(owner)/dashboard/page.tsx` | `"use client"` (439 lines) | Server component fetches data, passes to client children |
| `app/(owner)/dashboard/orders/page.tsx` | `"use client"` | Server component for data, client component for real-time updates |
| `app/(owner)/dashboard/analytics/page.tsx` | `"use client"` | Server component for data, dynamic import for Recharts |
| `app/(owner)/dashboard/settings/page.tsx` | `"use client"` | Server component fetches settings, client form component |
| `app/(owner)/dashboard/subscription/page.tsx` | `"use client"` | Server component for data display |
| `app/(owner)/dashboard/qr/page.tsx` | `"use client"` | Server component for QR generation, client for print |
| `app/(superadmin)/layout.tsx` | `"use client"` | Server layout + client ErrorBoundary island |
| `app/(customer)/layout.tsx` | `"use client"` | Server layout + client Navbar island |

**Pattern to follow:**
```
// Server component (page.tsx)
export default async function Page() {
  const data = await fetchData()
  return <ClientComponent data={data} />
}

// Client component (ClientComponent.tsx)
"use client"
export function ClientComponent({ data }) { ... }
```

### 2.2 Eliminate Waterfall Queries
**Current problem**: Every owner page runs 4-9 sequential client-side queries.

**Fix — Create shared data loader:**
- **New file**: `lib/loaders.ts` — Server-side data fetching functions
- **New file**: `app/(owner)/dashboard/layout.tsx` — Fetch `restaurantId` once, pass via layout props or React context
- Each page only fetches its specific data, not auth + restaurant + data sequentially

### 2.3 Unbounded API Queries
- **File**: `app/api/superadmin/customers/route.ts` — Change `.limit(20000)` to paginated (max 100)
- **File**: `app/api/superadmin/restaurants-stats/route.ts` — Same fix
- **File**: `app/api/superadmin/users/route.ts` — Paginate `listUsers()`
- Add `Cache-Control` headers to public endpoints (menu, settings)
